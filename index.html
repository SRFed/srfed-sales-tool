📄 Opening your documents in Word
1. Click “Enable Editing.” Word opens any file downloaded from a browser in Protected View. This is a Windows/Office security feature (not something the file can switch off), so this one click is required on every volume.
2. Update the Table of Contents. These files are set to refresh fields when they open, so after you enable editing Word usually asks “This document contains fields that may refer to other files. Do you want to update the fields?” — click Yes and the Table of Contents and its page numbers fill in. (That field-update prompt is separate from the Protected-View bar in step 1.)
3. If you don’t see that prompt — or the page numbers still read “1” — click anywhere in the Table of Contents and press F9 (or right-click → “Update Field” → “Update entire table”). The header, solicitation number, and company fields are already filled in.

  1. In HubSpot, click the Settings gear (top right)

A weak critical factor (technical capability, past performance, scope clarity) caps the fit — one real gap is decisive, not averaged away. When that ceiling is binding, the score stops there: spread calibration is skipped so a genuine gap isn’t penalized twice. 🔐 Microsoft 365 Sign-In (Azure)

Connects the tool to a Microsoft 365 organization for sign-in, SharePoint, and document conversion. These come from an Azure App Registration (Azure Portal → App registrations). Changing them re-initializes sign-in — you'll be asked to reload.

Tenant ID: use your organization's GUID to restrict sign-in to your org only, or organizations to let any Microsoft 365 business tenant sign in (for multi-tenant deployments).
Authorized domain: restricts who can generate proposals. Enter one domain, several (comma-separated), or leave blank to allow any signed-in account.
Redirect URI: must exactly match a redirect URI registered in the Azure app, and is normally the URL where this tool is hosted.
